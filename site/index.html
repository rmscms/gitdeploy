GitDeployPro 1.1.0

Backup Scheduler & Database Manager Refresh

Release 1.1.0 focuses on reliability: a fully monitored backup scheduler, fast import/export parity with mysqldump, and a database workspace that handles production-scale datasets without timeouts.

Feature Highlights

Backup Scheduler Control

  • Scheduler now autostarts on app launch when enabled in Settings and recovers missed jobs.
  • Main toolbar shows live countdown for the next run when no jobs are active.
  • New Active Tasks button opens a modal with per-task row progress, current table, and a Stop action.
  • Live log streaming mirrors “Run now” so background jobs report tables processed, batches, health, and output folders.
  • Failures trigger Windows notifications even while minimized.

mysqldump-Compatible Backups

  • Dump header/footer now preserves SET NAMES, sql_mode, time zones, charset, collation, unique/foreign key flags, and notes.
  • Table exports wrap LOCK TABLES / ALTER TABLE … DISABLE|ENABLE KEYS and batch inserts for speed.
  • All data types use MySQL’s escaping rules, including strict DateTime parsing/formatting for cross-version imports.
  • Supports zip and tar.gz outputs with health summaries plus total backup counters.

Global Configuration

  • All scheduler plans and history migrate to global_config.json under AppData for roaming persistence.
  • Backups can be launched from anywhere while sharing the same queue/history across projects.
  • Health logs survive reinstalls and new workspace clones.

Database Manager Upgrades

  • Fast imports for .sql, .zip, .tar.gz with create/drop prompts and live metrics (bytes, statements, rows).
  • Configurable command timeout plus “rows affected” telemetry for heavy queries.
  • ComboBox search filters locally with instant dropdown and zero extra queries.
  • Context actions add “Truncate table” and “Empty table” for maintenance.
  • Import pipeline normalizes unsupported sql_mode flags and offers “Continue on errors”.

Changelog 1.1.0

  • Added scheduler auto-start toggle plus missed-run recovery dialog.
  • Implemented BackupTaskMonitor window with live activity stream, cancel buttons, row progress, and history.
  • Moved schedules/history to roaming global_config.json for project-agnostic persistence.
  • Standardized DatabaseBackupService output to mirror mysqldump headers, session settings, locking, and batched inserts.
  • Introduced zip/tar.gz exports, open-folder shortcuts, per-plan backup counters, and health snapshots.
  • Database Manager imports now support archives, decompression, DB create/drop, progress bars, and optional continue-on-error.
  • Table search is now client-side with automatic dropdown expansion and no redundant queries.
  • New table maintenance menu exposes “Truncate” and “Empty” commands using the active connection.
  • Command timeout selector, run log rows-affected counter, and improved notifications for failures.
  • NotificationService now marshals toast UI through the WPF dispatcher for stability.
